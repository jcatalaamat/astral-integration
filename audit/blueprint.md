# audit: /blueprint (BlueprintPage)

## verdict: WRONG DESIGN SYSTEM + part of old service model.

### critical issue: old design system
- studio-bg, rectangular buttons, no rounded corners
- visually inconsistent with rest of site

### content issues

1. **priced at €2,500** — the current site positions everything as "$0 upfront, % rev share." a €2,500 deliverable is a fundamentally different business model. both can coexist but they need to be framed correctly. currently they just contradict.

2. **references "Digital Realignment Review" as prerequisite** — "Following a completed Review" (€450). so the full path is: review (€450) > blueprint (€2,500) > build (?). total cost before any building: €2,950. this directly contradicts "no upfront cost."

3. **the content is good** — "what this is" / "what it clarifies" / "how it works" / "what you receive" / "what this is not" structure is clear and well-reasoned. the writing is better than some of the modern pages. but it belongs to a different business model.

4. **CTA "Start with a Review"** — links to /review, perpetuating the old-design chain

### recommendation
- **decide**: is the review+blueprint model still offered alongside the rev-share model?
  - if YES: redesign these pages in the current design system, and clearly explain how both models work on /pricing
  - if NO: remove /blueprint, /review, /start-here, /process, /studio-collaboration pages
- either way, having them publicly accessible in an old design system while the main site looks completely different is damaging to brand trust
